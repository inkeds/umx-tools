# 六大组合选型矩阵

## 固定前提

无论选择哪个组合，Vibe 文档都必须落实：Epic -> Feature/Story -> Spec + 里程碑。

## 组合定义

| 编号 | 组合 | 适用场景 |
|---|---|---|
| c1 | 需求画布 -> 原型 -> AI 生成 | 快速起步、小工具 |
| c2 | Story Mapping -> 任务拆解 -> AI 迭代 | 中小项目持续迭代 |
| c3 | 场景驱动 -> 接口定义 -> 前后端并行 | 前后端分离、联调优先 |
| c4 | Figma -> Prompt -> AI 全栈 | 设计驱动产品 |
| c5 | 精益 MVP -> 反馈 -> AI 快迭代 | 想法验证、快速试错 |
| c6 | DDD 精简 -> 骨架 -> AI 填充 | 复杂业务、长期维护 |

## 推荐起步

- 默认：`auto + single-file`
- 需要接口协作：`c3 + minimal`
- 复杂域治理：`c6 + standard/full`
